7  AI工具使用说明

7.1 工具配置

本次研究主要使用两款AI工具：

工具 定位 使用场景
Claude Code（VSCode插件） 主力的代码理解和任务完成工具 全程使用，包括代码编写、报错分析、修改调试
ChatGPT 辅助、补充 概念答疑、部分代码参考

Claude Code以项目”司机”的角色主导整个研究流程，ChatGPT主要用于事后补充验证。

7.2 Claude Code使用流程

7.2.1 提问框架

在提问前先梳理清楚以下几个要素，描述得越清楚，得到的答案越精准：

要素 说明 示例
身份 你是谁、研究背景 “我是金融学学生，在做A股Beta系数研究的课程报告”
场景 当前遇到什么问题 “本地已有股票收益率数据，存储在returns.csv中”
要求 你想要什么 “想用CAPM模型回归估计每只股票的Beta”
输出 期望得到什么形式 “输出一个包含股票代码和Beta值的DataFrame”
约束 有没有额外限制 “不能使用除了akshare、pandas、statsmodels以外的库”

完整示例：

我是一名中山大学金融专属学生，正在上金融数据分析与建模的课程，需要完成一个A股Beta系数研究的课程报告作业。

【已有】股票日收益率数据保存在data目录下，包含5只股票和沪深300指数的收益率序列。

【目标】用CAPM模型$$  r_i - r_f = α + β*(r_m - r_f) + ε $$进行回归，估计每只股票的Beta系数。

【期望输出】一个包含股票代码、Beta值、t统计量、R方的结果表格。

【约束】使用statsmodels进行OLS回归，不要用其他计量库。

7.2.2 典型工作场景

代码编写: 描述功能需求，Claude Code生成完整代码框架

报错处理: 粘贴报错信息，Claude Code分析原因并给出修复建议

代码修改: 说明修改意图，Claude Code直接改动对应文件

7.2.3 有效使用心得

说清楚输入输出: 描述任务时说明数据来源和期望结果，代码质量会明显提升

分步操作: 复杂任务拆成多个小步骤，比一次性让AI写完整代码效果更好

主动质疑: 当Claude Code的建议不符合预期时，直接指出，它会调整方向

Warning

保留判断力: 对金融理论和模型假设保持清醒，AI负责执行，不负责替你做判断

7.3 总结

Claude Code是本次研究的主要生产力工具，负责代码编写、报错排查、逻辑验证全流程。Chat GPT作为补充，在概念理解和代码参考层面提供辅助。AI提升的是研究效率，但金融逻辑和模型判断仍需自己把控。
